Donkeycraft — Test Suite

Functional tests for each module group. Click a test to open it in your browser. Entries follow PLAN.md's dependency order.
1. Core Infrastructure
Tests for all 9 core modules (~2,308 lines): namespace, EventBus, config, logger, timer, input, math-utils, audio, init-sequence — including event routing, delta-time accumulation, Perlin noise, strict-mode compliance, and IIFE namespace isolation
2. Asset Generation
Tests for the asset generator module (~4,732 lines): procedural texture generation using Simplex noise for 285+ block types, procedural sound generation via Web Audio API across 13 categories, and texture atlas canvas assembly
3. WebGL Rendering Engine
Tests for all 15 rendering modules (~3,404 lines): GL context creation, shader compilation, geometry builder, mesh optimizer, chunk mesh, terrain renderer, camera, fog, sky, lighting, hand renderer, break particles, GUI renderer, weather system — covering WebGL 1.0 constraints and batched draw calls
4. Block System
Tests for all 6 block modules (~2,436 lines): Block definitions (257 vanilla blocks with hardness/blast resistance/drop), BlockState variants, BlockTypes classification, TextureAtlas UV mapping, BlockModels with AO data, RecipeRegistry with crafting/furnace/shapeless matching — including cross-module integration
5. World Generation & Management
Tests for all 17 world modules (~4,323 lines): Chunk data structures (16×256×16), ChunkManager radius tracking, biome definitions, terrain generation via Perlin noise, ore distribution, 3D cave generation, water placement, surface layer per biome, lighting engine BFS flood-fill, physics (gravity blocks, liquid flow), dimension system (Overworld/Nether/End), portal detection, nether/end generators, world time (24000-tick day cycle), and utilities
6. Player System
Tests for all 10 player modules (~2,664 lines): Player entity (position, velocity, dimensions 1.8×0.6), Movement physics (walking/sprinting/swimming/flying), AABB collision detection & response, Jump mechanics with cooldown, Creative/Spectator flying, HurtBox damage/knockback/fall-damage, GameMode (Survival/Creative/Spectator), Stats tracking, Hunger system (food/saturation/starvation), Experience/XP levels
7. Interaction System
Tests for all 4 interaction modules (~1,283 lines): DDA voxel raycasting with hit detection and face normal, BlockAction breaking mechanics with tool speed multipliers and drop spawning, BlockPlacement with face normal handling and player AABB collision, InteractableBlocks right-click interactions (doors, chests, furnaces, levers, buttons, dispensers)
8. Inventory & GUI System
Tests for all 13 UI modules (~4,175 lines): ItemStack (count, NBT tags, stacking), Inventory (slots, drag state, shift-click), GuiManager (modal stacking, keyboard/mouse routing), Hotbar (slot rendering, number keys 1-9, scroll wheel), CraftingGrid (3×3 + output matching), CreativeInventory (6 tabs, search by name/ID), FurnaceUI (input/fuel/output/progress), ChestUI (27/54 slots), AnvilUI (rename/repair/enchant), EnchantingUI (random options, level cost), DebugOverlay (F3 screen), GuiElements (drag-drop, tabs, buttons, text input), LoadingScreen (progress bar, rotating tips)
9. Entity & Mob System
Tests for all 9 entity modules (~2,779 lines): Entity base class (position, velocity, health, bounding box, serialization), EntityManager (spawn/despawn/tick/by-type queries), PassiveMobs (cow/pig/sheep/chicken — wander/flee/item drops), HostileMobs (zombie/skeleton/spider/creeper/enderman — chase/attack/explode), BossMobs (Ender Dragon/Wither — phases/attacks/death animation), MobAI (A* pathfinding, line-of-sight, chase/flee/wander), Projectiles (arrows/snowballs/ender pearls/dragon breath — gravity/impact), Animals (breeding/babies/leads/food), MobSpawning (chunk/light/biome checks, mob caps)
10. Redstone System
Tests for all 6 redstone modules (~2,177 lines): RedstoneEngine tick scheduling and dirty queue, Wiring signal propagation with branching and torch states (strength 0–15), Repeater/Comparator (delays 1–4 ticks, compare/difference modes), Observers (facing detection, 1-tick pulse, cooldown), Pistons & sticky pistons (push chains up to 12 blocks, pull, crush detection), TNT (40-tick fuse, radius-8 explosions, blast resistance)
11. Game Systems
Tests for all 3 game system modules (~1,267 lines): EnchantmentRegistry (26+ vanilla enchantments, compatibility rules, application logic), PotionRegistry (40 potions, 18 effects, brewing recipes), ActivePotionManager (entity effect lifecycle), ToolRegistry (7 material tiers, speed multipliers, correct-for-drop detection, durability)
12. Storage System
Tests for all 3 storage modules (~1,530 lines): WorldStore (IndexedDB save/load, dirty chunk batching, quota handling, chunk format normalization), LevelData (spawn position, game mode, time, seed, player data, validation, auto-save with WorldStore integration), AssetCache (IndexedDB persistent caching for texture atlases and sounds, quota management)
13. Game System (Main Entry Point)
Tests for Donkeycraft.Game (~1,195 lines): constructor, state transitions (init → running → paused), movement controls, flying mechanics, interaction cooldowns, block overlap detection, hotbar selection, time of day sync, canvas resize handling, destroy cleanup, block retrieval, EventBus integration, pointer lock, camera-player synchronization
